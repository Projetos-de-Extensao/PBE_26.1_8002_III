## 🚀 Introdução

<p align="justify">
    O 5W2H é uma ferramenta administrativa que pode ser utilizada para diferentes contextos dentro de uma organização, como no planejamento estratégico e como guia na execução de ações. O 5W2H baseia-se em 7 perguntas fundamentais (What?, Why?, Where?, Who?, When?, How? e How much?), com o intuito de esclarecer e organizar ideias. No contexto deste projeto, a ferramenta será utilizada como um mapa diretivo, tornando os objetivos arquiteturais e de negócio evidentes para a equipe, auxiliando na compreensão do escopo e nas tomadas de decisão.
</p>

## 🛠️ Metodologia

<p align="justify">
    A equipe de desenvolvimento realizou reuniões de levantamento de requisitos e brainstorming para compreender as dores do fluxo atual de estágios da instituição. A partir das tecnologias selecionadas (como automação via IA e frameworks web) e dos processos mapeados, foi possível detalhar as questões do 5W2H com foco em engenharia de software.
</p>

## ❓ Questões 5W2H

| Questão | Resposta |
| :--- | :--- |
| **What?** <br>*(O que será feito?)* | Desenvolvimento de uma **Plataforma de Gestão de Estágios**, composta por uma API Backend e um portal Web, que integra Inteligência Artificial para a pré-análise de contratos (TCE) e relatórios de horas. |
| **Why?** <br>*(Por que será feito?)* | Para modernizar o fluxo de aprovação da instituição, eliminando o trabalho manual e repetitivo da Secretaria na conferência de dados básicos, acelerando o tempo de resposta para os alunos e garantindo que 100% dos documentos estejam em conformidade com a legislação acadêmica vigente. |
| **Where?** <br>*(Onde será feito?)* | A plataforma terá arquitetura em nuvem (Cloud). Será acessível via navegador web de qualquer dispositivo pelos alunos, e através de um painel de Back-office seguro para a Secretaria e Coordenação. |
| **When?** <br>*(Quando será feito?)* | O projeto será desenvolvido ao longo do período letivo de 2026.1, com a entrega da versão final prevista para **09/07/2026**. |
| **Who?** <br>*(Quem irá fazer / participar?)* | **Equipe de Desenvolvimento:** Lucas Jesus, Otto Balestrassi, Caio Ximenes, Daniel Alberto e Bernardo Miller.<br>**Atores do Sistema:** Alunos, Secretaria (Gestão Burocrática) e Coordenação (Gestão Pedagógica). |
| **How?** <br>*(Como será feito?)* | Utilizando Python com o framework **Django** para a construção da API (Controllers e Services), banco de dados relacional para persistência do histórico, e integração com IA para extração e validação de regras de negócio em arquivos PDF. O versionamento do código será via GitHub e a documentação via MkDocs. |
| **How much?** <br>*(Quanto custará?)* | Tratando-se de um projeto acadêmico, o custo principal é mensurado em horas de esforço de engenharia da equipe. Financeiramente, haverá custos de infraestrutura Cloud (hospedagem da aplicação e banco de dados), storage (armazenamento dos PDFs) e tarifação de consumo da API de Inteligência Artificial. |

## ✅ Conclusão

<p align="justify">
    O 5W2H provou ser uma técnica simples, porém poderosa. Dentro da disciplina, ela forneceu à equipe um alinhamento claro sobre o escopo do software, dividindo a visão macro em responsabilidades técnicas, justificativas de negócio e organização das entregas, garantindo que todos os membros caminhem na mesma direção.
</p>
